# User Journeys

## Journey 1: First-time user creates profile
1. User signs up
2. User uploads CV
3. System extracts structured profile
4. User reviews and confirms canonical candidate profile
5. Profile is saved for future tailoring

## Journey 2: User tailors for a job
1. User opens tailoring page
2. User selects profile
3. User pastes job description
4. System checks role fit
5. System shows fit summary and warning if needed
6. User chooses language and writing level
7. System generates tailored CV and cover letter
8. User rates output

## Journey 3: Weak-fit role
1. User pastes a job
2. System detects mismatch
3. Warning banner appears
4. User can continue or stop
5. Output is still allowed for MVP, but warning is logged

## Journey 4: Returning user
1. User logs in
2. Previous CVs, letters, and profiles are available
3. User can regenerate or create new versions faster

## Journey 5: Admin learning loop
1. Feedback is stored
2. Low-rated outputs are reviewed
3. Prompt and UI improvements are made
4. Test cases are updated
